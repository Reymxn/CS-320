CS 320 Portfolio Reflection

Throughout this course, I learned that creating functional and secure software requires careful planning, testing, and validation. I used unit tests with JUnit to verify that my code worked correctly and to catch errors before they became larger problems. Testing individual methods and services helped ensure the software behaved as expected while making future changes easier and safer.

When designing software, I start by identifying the project requirements and breaking them into smaller, manageable tasks. I focus on creating classes with clear responsibilities and writing code that is organized, readable, and maintainable. I also consider how users will interact with the program so that it meets their needs while remaining reliable and efficient.

To incorporate user needs into a program, I use the project requirements as a guide throughout development. I make sure each feature supports the intended functionality and verify that the implementation satisfies those requirements through testing. This approach helped me build software that was both functional and dependable while improving my understanding of software testing, quality assurance, and object-oriented design.
